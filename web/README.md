# web/

Vite + React + TypeScript implementation of the Islamic On This Day web
reading surface — the desktop "today" page that surfaces one verified
moment from Islamic history in both Hijri and Gregorian calendars.

The visual design is the editorial-architectural direction translated from
the Claude Design HTML/JSX prototype (see `Islamic On This Day.html`). The
mobile screens in the same prototype are intended for the Flutter app and
are not implemented here.

## Stack — mirrors Majlisna / LaTabdhir

| Concern | Choice |
|---|---|
| Package manager | **Bun** |
| Bundler | **Vite 6** |
| Language | **TypeScript 5** (strict) |
| Framework | **React 18** |
| Routing | **TanStack Router** (file-based, code-split) |
| Server state | **TanStack Query** |
| HTTP | **ky** (with bearer-token interceptor) |
| Styling | **Tailwind CSS v4** + `cn()` helper |
| Linting | **oxlint** |
| Testing | **Vitest** + Testing Library |
| Path alias | `@/*` → `./src/*` |
| Dev proxy | `/api` → `http://127.0.0.1:5111` (the FastAPI backend) |

## Run

```sh
cd web
bun install
bun run dev          # http://localhost:3000
bun run build        # production bundle in dist/
bun run preview      # serve the production build
bun run check        # typecheck + lint + tests
bun run test         # vitest run
bun run test:watch   # vitest --watch
```

## Design language — what to preserve when extending

- **Type stack** — Cormorant Garamond (serif headlines + body), Inter Tight
  (UI/sans), JetBrains Mono (eyebrows, captions, metadata), Amiri (Arabic).
- **Color** — warm cream paper / deep ink-blue dark mode. Single accent: a
  muted oxidised-copper / verdigris (`oklch(58% 0.06 165)`) used only for
  small architectural marks and verification chips. No gradients, no glow.
- **Frieze language** — the architectural section divider is the system's
  cohesion device. Heavy top rule + hairline below + small geometric rosette
  centred. Use `FriezeRule` for any new section break.
- **Image policy** — when an event has no image (`noImage: true` in data),
  render `NoImagePlate`, never an empty slot.
- **Disputes are visible** — when `event.disputed` is true, surface the
  `DisputedDrawer`. Every position is weighted equally.

The reader components use **inline styles** for the typographic shell on
purpose — the design lives on precise px values (132px Hijri-day numeral,
letter-spacing -4, oklch tokens) that don't survive utility-class
abstraction cleanly. **Tailwind is wired** for new components (modals,
forms, controls) where utility classes pay off. Use `cn()` from
`@/lib/utils` to compose them.

## Talking to the backend

The `vite.config.ts` proxies `/api` → `http://127.0.0.1:5111`, which is
where the FastAPI service in `backend/` listens by default. The seed in
`src/data/seed.ts` mirrors the `/api/v1/today` response shape, so swapping
to a `useQuery(...)` against the real endpoint is a small change.

## Standing rules (mirrored from Majlisna / LaTabdhir)

- TypeScript strict on (no implicit `any`, no unused locals, no fallthrough).
- Imports at the top of the file, no nested function definitions, early
  returns for guard clauses.
- TanStack Query owns all server state. Do not introduce Redux / Zustand.
- One source per module; group by domain folder, not by type.
