# Islamic On-This-Day тАФ Web client

Vite + React + TanStack Router reading surface for the Islamic On-This-Day
project. Trilingual (en / fr / ar with RTL), light/dark theme via CSS
variables, custom editorial design system, TanStack Query backed by an
auto-generated [`@hey-api/openapi-ts`](https://github.com/hey-api/openapi-ts)
client.

## Stack

| Concern | Choice |
|---|---|
| Package manager | **Bun** |
| Bundler | **Vite 6** (esbuild minify, vendor `manualChunks`) |
| Language | **TypeScript 5** (strict) |
| Framework | **React 18** |
| Routing | **TanStack Router** (file-based, code-split) |
| Server state | **TanStack Query** |
| HTTP | **`@hey-api/client-fetch`** (the only client тАФ no `ky`, no `axios`) |
| API codegen | **`@hey-api/openapi-ts`** (regenerated from `web/openapi.json`) |
| Styling | **Tailwind CSS v4** with **CSS variables** (`@theme inline`) |
| Theme | `.dark` class on `<html>` (toggled by `ThemeProvider`) |
| i18n | **i18next** + lazy-loaded JSON locales, `react-i18next` hooks |
| A11y primitives | **`@radix-ui/react-dialog`** for the disputed-views drawer |
| Errors | **`@sentry/react`** (gated on `VITE_SENTRY_DSN`) |
| Linting | **oxlint** |
| Formatting | **oxfmt** |
| Testing | **Vitest** + Testing Library + happy-dom |
| Path alias | `@/*` тЖТ `./src/*` |
| Dev proxy | `/api` тЖТ `http://127.0.0.1:5111` |

## Design system

The project ships **custom Tailwind components, not shadcn/ui**. The
editorial visual language (frieze rosettes, eight-point stars, dot-chip
verification chips, mono-uppercase eyebrows, paper-cream palette) is
bespoke enough that shadcn's defaults would have been re-themed top to
bottom. Radix primitives are pulled in *selectively* for a11y-critical
interactive components only тАФ currently `@radix-ui/react-dialog` under
`DisputedDrawer`.

**Tokens live in `src/index.css`** as CSS custom properties on `:root`
and `.dark`. The `@theme inline` block re-exports them as Tailwind colour
/ font scales so utilities like `text-ink`, `bg-paper`, `font-mono` read
the same vars. Dark mode flips via `<html class="dark">` тАФ no component
branches on it. To add a new colour token, add it to both `:root` and
`.dark` and to the `@theme inline` map.

### Design language тАФ what to preserve when extending

- **Type stack** тАФ Cormorant Garamond (serif headlines + body), Inter
  Tight (UI/sans), JetBrains Mono (eyebrows, captions, metadata), Amiri
  (Arabic).
- **Colour** тАФ warm cream paper (`#F5F0E6`) / deep ink-blue dark
  (`#0F1217`). Single accent: a muted oxidised-copper / verdigris
  (`oklch(58% 0.06 165)`) used only for small architectural marks and
  verification chips. No gradients, no glow.
- **Frieze language** тАФ the architectural section divider is the
  system's cohesion device. Heavy top rule + hairline below + small
  geometric rosette centred. Use `FriezeRule` for any new section break.
- **Image policy** тАФ when an event has no image (`noImage: true` in
  data), render `NoImagePlate`, never an empty slot.
- **Disputes are visible** тАФ when `event.disputed` is true, surface the
  `DisputedDrawer`. Every classical position is weighted explicitly.

## API client lifecycle

The OpenAPI client is generated from `web/openapi.json`. **Never edit
`src/api/generated/`.** Regenerate after backend schema changes:

```sh
# from backend/
uv run python -c "import json; from iotd.app import create_app; print(json.dumps(create_app().openapi()))" \
  > openapi.json && cp openapi.json ../web/openapi.json
# from web/
bun run generate-api
```

Hand-rolled hooks in `src/api/{today,events,lessons,тАж}.ts` are thin
wrappers around the generated `*Options` helpers тАФ they exist to expose
`enabled` toggles and to centralise per-route defaults. The query-key
map in `src/api/queryKeys.ts` is the source of truth for invalidation:
reach for it instead of re-deriving the option object at the callsite.

## Conventions

1. **Custom Tailwind, not shadcn.** Visual fidelity comes first тАФ the
   project's editorial design isn't shadcn-shaped.
2. **CSS variables, not JS branching.** Theme switching happens once at
   the root; nothing reads `theme === "dark"` except `ThemeProvider`.
3. **i18next for UI strings.** `useTranslation()`'s `t(...)` for chrome,
   `pickLocalised(...)` for content from the API payload (different
   system, different concern).
4. **No `ky`, no second HTTP client.** When auth lands, configure the
   generated client via `client.setConfig(...)` or interceptors.
5. **Routes have zero per-page chrome duplication.** Use `DetailHeader`
   for *-detail pages; use `PageShell` for landing pages with a subtitle
   and Footer.
6. **Loading / Empty / NotFound / ErrorBoundary live in `components/ui/`.**
   Don't reinvent per-route variants.
7. **Hijri month names** come from `@/i18n/months.ts` тАФ never redeclare
   the array in a route file.
